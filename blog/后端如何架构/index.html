后端如何架构

2021年9月13日 · 阅读需 1 分钟

MVC（Model-View-Controller）​

这个架构设计简单明了。 从路由接口访问，首先访问 C 控制层。

  • Controller（控制器）：处理 HTTP 请求，调用 Model，返回结果给前端。

    • 控制层主要负责，比如验证身份，验证请求参数是否符合等等

  • Model（模型）：数据层，负责与数据库交互

    • 模型层主要负责，定义数据结构，操作数据库，返回结果。

  • View（视图）：表现层，通常是前端页面（在 API 服务器中可能不会有 View）。
