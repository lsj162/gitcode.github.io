后端如何架构

2021年9月13日 · 阅读需 1 分钟

MVC（Model-View-Controller）​

这个架构设计简单明了。 从路由接口访问，首先访问 C 控制层。

  • Controller（控制器）：处理 HTTP 请求，调用 Model，返回结果给前端。

    • 控制层主要负责，比如验证身份，验证请求参数是否符合等等

  • Model（模型）：数据层，负责与数据库交互

    • 模型层主要负责，定义数据结构，操作数据库，返回结果。

  • View（视图）：表现层，通常是前端页面（在 API 服务器中可能不会有 View）。

添加评论​

使用 giscus 实现评论功能。https://giscus.app 完成上面三步骤，即可验证。

安装​

pnpm add @giscus/vue

配置​

<!-- /components/Comments.vue -->

<template>
  <Giscus
    v-if="showComment"
    :repo="giscusConfig.repo"
    :repo-id="giscusConfig.repo_id"
    :category="giscusConfig.category"
    :category-id="giscusConfig.category_id"
    mapping="title"
    strict="0"
    reactions-enabled="1"
    emit-metadata="0"
    input-position="top"
    theme="light"
    lang="zh-CN"
    loading="lazy"
    crossorigin="anonymous"
    async
  />
</template>

<script lang="ts" setup>
import Giscus from "@giscus/vue";
import { useRoute } from "vitepress";
import { reactive, ref, watch } from "vue";
const route = useRoute();

const giscusConfig = reactive({
  repo: "...", // # 这里就是giscus官方提供的
  repo_id: "...", // # 这里就是giscus官方提供的
  category: "...", // # 这里就是giscus官方提供的
  category_id: "...", // # 这里就是giscus官方提供的
});

const showComment = ref(true);

watch(
  () => route.path,
  () => {
    showComment.value = false;

    setTimeout(() => {
      showComment.value = true;
    }, 200);
  },
  {
    immediate: true,
  }
);
</script>

添加到全局​

// https://vitepress.dev/guide/custom-theme
import Theme from "vitepress/theme";

export default {
  extends: Theme,

  Layout: () => {
    return h(Theme.Layout, null, {
      // https://vitepress.dev/guide/extending-default-theme#layout-slots
      "doc-after": () => h(Comments),
    });
  },
};

前端如何架构

2021年9月13日 · 阅读需 2 分钟

以电商平台为例。

1. 技术选型？​

首先我们可能考虑主流前沿技术，比如 react，vue。 然后搭建一个自动智能工具的开发环境。

我们考虑是否要支持 seo？​

这里又涉及到以上技术的技术框架选型。比如 nextjs，nuxtjs（当然还有很多其他可选）。

考虑搭建平台用户端系统？​

代码模块化实现业务，那么当业务独立是否考虑微服务。比如：用户端客服系统。统一登录注册中心系统。

考虑搭建平台后台管理系统？​

后端管理数据，管理员，审核等一系例操作系统。

开发提交测试？​

没有经过测试上线必然挂掉（100%可能）。那么测试技术工具又如何选择。比如 jest。还是人工肝。
