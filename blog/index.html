Postfix 基本配置

2025年4月10日 · 阅读需 1 分钟

介绍​

Postfix 有数百个配置参数，这些参数通过 main.cf 文件进行控制。幸运的是，所有参数都有合理的默认值。在许多情况下，您只需配置两三个参数，就可以开始使用邮件系统。

感兴趣的：

  • 外发邮件使用什么域名
  • 将邮件转发到哪些目的地
  • 从哪些客户端中继邮件

Postfix 配置文件​

默认情况下，两个最重要的文件是 main.cf 和 master.cf。

配置参数语法格式：

# /etc/postfix/main.cf：​
    参数 = 值
    其他参数 = $参数

Postfix 使用数据库文件进行访问控制、地址重写和其他用途。

# /etc/postfix/main.cf:
    virtual_alias_maps = hash:/etc/postfix/virtual

# postfix reload

socket

2025年4月6日 · 阅读需 2 分钟

Socket（套接字）是一种网络编程接口，它既不属于协议，也不属于软件，而是一种抽象层，用于在不同设备或进程之间进行数据传输。

Socket 的分类​

基于传输协议 ❓​

  • TCP Socket（流式套接字）：基于 TCP 协议，提供可靠的、面向连接的通信。数据传输保证顺序和完整性
  • UDP Socket（数据报套接字）：基于 UDP 协议，提供无连接的服务。数据传输不保证顺序和完整性，但速度更快
  • Raw Socket（原始套接字）：允许应用程序直接访问网络底层协议（如 ICMP）。适用于需要处理特殊网络报文的应用

基于地址族 ❓​

  • AF_INET：IPv4 地址族
  • AF_INET6：IPv6 地址族
  • AF_UNIX：用于进程间通信的 Unix 套接字

Socket 的工作原理​

服务器端：

- 创建 Socket：初始化监听端口。
- 绑定（bind）：将 Socket 绑定到指定 IP 和端口。
- 监听（listen）：等待客户端连接请求。
- 接受连接（accept）：与客户端建立连接。
- 数据收发（send/recv）：进行数据通信。
- 关闭连接（close）：释放资源

客户端：

- 创建 Socket。
- 连接服务器（connect）。
- 数据收发（send/recv）。
- 关闭连接（close）

Socket 的设计目的，应用场景​

  • 网页浏览：使用 TCP Socket 进行可靠的数据传输。
  • 实时音视频传输：使用 UDP Socket 保证传输速度和实时性。
  • 网络安全监控：使用 Raw Socket 进行底层网络数据包的捕获和分析

git

GIT 操作命令​

  • git config user.name [查看用户名]
  • git config user.email [邮箱地址]
  • git config --global user.name "username" [全局修改用户名]
  • git config --global user.email "email" [邮箱地址]
  • git config user.name "username"[局部]
  • git config user.email "email"[局部]

提交准备​

  • git init [初始化]
  • git add [放入暂存区]
  • git status [状态]
  • git commit -m [提交并说明]
  • git diff [查看修改内容]
  • git reset --hard commit_id [回退和恢复版本分支]
  • git checkout -- file [工作区的内容修改撤销]
  • rm || git rm [删除工作区||暂存区]

Github​

一旦提交推送到远程版本库，就形成记录 ……

  • ssh [密码]
  • git remote -v [远程库信息]
  • git remote add origin git@github.com:somewhile/systemjs.git [git 关联 github]
  • git branch --set-upstream-to=origin name [关联远程分支]
  • git remote rm origin [删除关联]
  • git push -u origin master [推送]
  • git pull [更新]
  • git clone --depth=1 [克隆且最近一次提交分支]
  • git stash [pull 冲突，本地暂时先存储起来]
  • git stash pop stash@0 [删除暂存]
  • git stash apply [恢复当前分支]

分支管理​

  • git checkout -b name 远程或本地 [创建分支]
  • git push origin [提交分支]
  • git branch [查看]
  • git checkout master [切换分支]
  • git merge dev [合并分支]
  • git branch -d||D dev [删除分支,不合并强制删除]
  • git push --set-upstream origin name [提交远程分支]
  • git push --delete origin name [删除远程分支]

数据序列化

基本例子 🌰​

---
#  : 代表对象
languages:
  # - 代表数组
  - Ruby
    Perl
  - Python
websites:
  date: 1976-07-31
  str: 这是一行字符串
  bool: true
  # 注释：~ 代表null
  parent: ~
  # 注释：!! 代表强制转换数据类型
  f: !!str true

# & 用来建立锚点（defaults）* 用来引用锚点 << 表示合并到当前
defaults: &defaults
  adapter: postgres
  host: localhost

development:
  database: myapp_development
  <<: *defaults
